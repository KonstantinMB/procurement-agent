 AI Procurement Officer — Hackathon Build Plan

 ▎ Working title (swap freely): "Atlas" — an autonomous AI procurement officer you brief in one sentence, that finds suppliers, calls and negotiates by voice live, emails RFQs, fills a real-time comparison dashboard, and closes the deal on one
 ▎ button.

 Context

 This is a hackathon project (stack: Claude Agent SDK). The buyer types a single request into a command bar — e.g. "I need 50 brushless motors delivered by Friday under €60/unit" — and a headless Claude Agent SDK loop (Opus 4.8, effort: max)
 autonomously runs a full procurement cycle while the UI shows what the agent is doing, live:

 1. Spawns sub-agents to search the web for suppliers (the "swarm").
 2. Places a real outbound AI phone call (via Vapi — Claude brain + ElevenLabs voice) to a supplier and negotiates price/lead-time live.
 3. Emails RFQs to other suppliers.
 4. Auto-populates a real-time RFQ comparison dashboard.
 5. Keeps a chat going and uses AskUserQuestion to ask the buyer for missing requirements.
 6. Ends with a master "Order Now" button (payment simulated — "fake it till you make it").

 Why: Validated demand exists (Pactum, Mandel AI — both funded) but nobody glues web-discovery + live voice negotiation + dashboard + one-click close into a single "AI employee." The voice call is the reliable jaw-drop. Goal: a flashy, reliable
 demo that makes judges go "wow."

 Decisions locked with the user

 - Real: Agent SDK orchestration · multi-subagent web search · live phone call (hero) · email RFQs · real-time dashboard · chat + AskUserQuestion · Order Now button.
 - Simulated: Stripe/payment + invoice settlement at the end (no real charge).
 - Stack: Vite + React + TypeScript + Tailwind (client); Hono (Node 24) backend running the Agent SDK; SSE for server→client streaming.
 - Auth: Agent SDK uses the already-authenticated claude CLI (no ANTHROPIC_API_KEY).
 - Keys user provides: Vapi (orchestrates the call), ElevenLabs (voice), Twilio (optional — only if BYO number into Vapi), OpenAI/Whisper (optional).
 - Time budget: treat as a full hackathon day; plan is tiered (demo-safe core → polish) so it degrades gracefully.

 Canonical event model (server → client, the contract for the whole UI)

 A discriminated union in server/src/events.ts, shared with the client:
 - agent.ready { model, apiKeySource }
 - agent.text_delta { text } · agent.thinking { active }
 - tool.call { id, name, label, input, status:"running" } · tool.result { id, status:"done"|"error", summary }
 - subagent.spawned { id, parentId, role } · subagent.done { id }
 - rfq.supplier_added { vendor } · rfq.supplier_updated { id, patch }  (status, initialPrice, negotiatedPrice, leadTimeDays)
 - rfq.summary { headline, savings, withinBudget }
 - call.ringing { supplierId } · call.connected · call.transcript { speaker, text } · call.quote { price, leadTimeDays } · call.ended { outcome }
 - email.sent { supplierId, to } · email.reply { supplierId, parsed }
 - question.ask { id, questions } · question.answered { id, answers }
 - order.placed · order.receipt { invoice }

 Why this matters: the dashboard is the agent's tool-calls, streamed. The agent drives the board by calling add_supplier / update_quote (handlers emit rfq.*); the SDK stream provides tool.call/result/text_delta for the activity rail. Build the
 UI against this contract + demo.ts replay first, then wire the real agent.

 Backend: Claude Agent SDK integration (grounded in current docs)

 Package: @anthropic-ai/claude-agent-sdk (v0.3.x, Node ≥18). Docs: code.claude.com/docs/en/agent-sdk/typescript.

 const q = query({
   prompt: inbox.stream(),                 // streaming-input async generator (continuous chat)
   options: {
     model: "claude-opus-4-8",
     effort: "max",                        // first-class option; thinking stays adaptive
     thinking: { type: "adaptive" },
     includePartialMessages: true,         // text deltas → SSE
     settingSources: [],                   // controlled backend; don't auto-load FS config
     systemPrompt: PROCUREMENT_SYSTEM_PROMPT,
     mcpServers: { app: appServer },       // in-process custom tools
     allowedTools: ["mcp__app__*", "WebSearch", "WebFetch", "Agent", "AskUserQuestion"],
     agents: SUBAGENTS,                    // supplier-scout ×N
     canUseTool,                           // AskUserQuestion → UI; auto-allow our tools
     maxTurns: 60,
     maxBudgetUsd: 5,                      // safety rail
   },
 });
 - Continuous chat: keep one long session; push new user messages into an async push-queue (inbox) yielded as SDKUserMessage; call q.interrupt() to make the agent react immediately to a steering message. (Report §7.)
 - Streaming → events: iterate q; map system/init→agent.ready, stream_event text_delta→agent.text_delta, assistant tool_use→tool.call, user tool_result→tool.result, result→done. Tag anything with parent_tool_use_id as subagent activity. (Report
 §6.)
 - Custom tools: createSdkMcpServer({ name:"app", tools:[ tool("add_supplier", …, zodSchema, handler), … ] }); each handler does the real work and emits a bus event. Reference via mcp__app__<tool>. (Report §5.)
 - AskUserQuestion: built-in; intercept in canUseTool when toolName==="AskUserQuestion" → render choices in UI → return the user's pick in updatedInput.answers. (Report §8.)
 - Auth: no API key in env → SDK uses logged-in claude creds automatically. For robustness/containers: claude setup-token once → set CLAUDE_CODE_OAUTH_TOKEN. (Contractual note: claude.ai-login is fine for an internal/demo build; a shipped
 product for external users would need ANTHROPIC_API_KEY. Report §4.)

 Subagent "swarm": define supplier-scout (description, prompt = "search the web for suppliers of , return vendors w/ contact", tools: ["WebSearch","WebFetch","mcp__app__add_supplier"], model:"opus" or "sonnet", effort:"high"). Orchestrator
 spawns several in parallel (per region/category) via the Agent tool → multiple lanes light up in the activity rail at once. Each scout calls add_supplier → cards stream in.

 ---
 Telephony: the live phone call (hero) — via Vapi (verified against docs.vapi.ai)

 Chosen path: Vapi orchestrates the whole real-time pipeline (telephony, STT, turn-taking, barge-in, streaming TTS); we keep Claude as the call brain and ElevenLabs as the voice. Use @vapi-ai/server-sdk to place calls + create the assistant;
 handle webhooks as raw Hono (parse JSON, import SDK types only). Cost: ~free ($10 signup credit; all-in ≈ $0.15–0.40/min — budget test calls).

 - Vapi assistant (create once, reuse by id): vapi.assistants.create({ model:{ provider:'anthropic', model:<claude id>, messages:[{role:'system', content: NEGOTIATION_PROMPT with {{vars}}}], tools:[report_quote] }, voice:{ provider:'11labs',
 voiceId }, firstMessage, analysisPlan:{ structuredDataSchema } }).
   - Claude model: Vapi's Anthropic list lags Anthropic releases — confirmed-good claude-sonnet-4-20250514 (or claude-opus-4-20250514); pick the newest Claude shown in the dashboard at build time. (This is the call brain; our orchestrator stays
 claude-opus-4-8.)
   - Keys: ElevenLabs is BYO — add ELEVENLABS_API_KEY in Vapi Dashboard → Integrations (voiceId resolves after). Anthropic can be Vapi-managed (pay via credits — fastest) or BYO.
   - Structured quote at call end: set analysisPlan.structuredDataSchema ({ unitPrice, currency, leadTimeDays, agreed }) so the end-of-call report returns a parsed quote (analysis is computed a few seconds after hangup).
 - report_quote function tool: { type:'function', function:{ name:'report_quote', parameters:{ unitPrice, leadTimeDays, moq?, notes? } }, server:{ url:<our>/webhooks/vapi } }. Mid-call Vapi POSTs a tool-calls message; our server must reply with
 the exact shape { results:[{ toolCallId, result:"<string>" }] } (else the model hangs).
 - call_supplier tool (agent-invoked): vapi.calls.create({ assistant:{ assistantId }, phoneNumberId, customer:{ number }, assistantOverrides:{ variableValues:{ supplier, part, qty, target_price, walk_away, lead_time } } }). Emit call.ringing
 immediately; keep a Map<call.id, resolve> of pending calls; resolve the tool's promise on end-of-call-report (return transcript + structured quote), using the live report_quote args as a fallback.
 - Webhooks (Vapi → POST /webhooks/vapi, verify x-vapi-secret): status-update (queued/ringing/in-progress/ended → call.*) · transcript (role + partial/final → call.transcript) · tool-calls report_quote → call.quote + update_quote (and reply
 with results) · end-of-call-report (artifact.transcript/messages/recordingUrl + analysis.summary/structuredData → resolve the call promise). Host the webhook near us-west-2; respond < ~6s.
 - Number: Vapi-provisioned US number (POST /phone-number {provider:'vapi', numberDesiredAreaCode}) — no Twilio trial friction, but US-only + a daily free-call cap; BYO Twilio for non-US or volume.
 - Negotiation policy lives in the assistant's system prompt (Claude reacts per turn); the orchestrator only sets the envelope via variableValues.
 - Demo fallbacks (pre-stage): call a teammate's phone as "the supplier"; keep a pre-recorded two-voice audio + canned call.quote played through the CallPanel (visually identical); do one real test call before stage.
 - Backend deps: @vapi-ai/server-sdk (calls + types); raw Hono for the webhook receiver.

 1. Visual identity

 Light, professional, white-surface. White cards on a soft off-white app background; hairline gray borders; ink text; one confident brand accent (blue) for primary actions + the active nav item; status colors used sparingly. Fonts: Geist (UI) +
 Geist Mono (all data — prices, lead-times, supplier names, the activity log; mono-on-data keeps the "instrument" precision even on white). Depth on white = hairline borders + soft layered shadows (no heavy drop-shadows, no glassmorphism);
 optional faint dotted-grid behind the swarm graph. Motion = near-critically-damped springs, never bouncy; honor useReducedMotion().

 Hard anti-slop rules: NO indigo→purple gradients; NO glassmorphism; NO three-icon marketing grid; NO pure #000 on pure #fff (ink #0F172A on #FFFFFF/#F7F8FA); one accent per region; color only on active/live state + primary action.
